1. Use Case Diagram
Main Actors and Key Use Cases:
I identified three main actors: Students who take courses and earn certificates, Instructors who create and manage courses, and Admins who oversee the platform and approve content.
Brief Description of Each Use Case:
UC1 - Browse Courses: Students view available courses on the platform.
UC2 - Enroll in Course: Students register for a course to access its content.
UC3 - Complete Lesson: Students finish lesson materials and mark them done.
UC4 - Take Quiz: Students answer questions and get graded automatically.
UC5 - View Certificate: Students access their earned certificates.
UC6 - Track Progress: Students check their completion stats and grades.
UC7 - Create Course: Instructors design new courses.
UC8 - Manage Course: Instructors edit and update their courses.
UC9 - Upload Lesson: Instructors add lesson content to courses.
UC10 - Design Quiz: Instructors create quizzes for assessment.
UC11 - Add Questions: Instructors write quiz questions.
UC12 - Manage Users: Admins control user accounts.
UC13 - Approve Course: Admins review and approve new courses.
UC14 - Remove Course: Admins delete bad or outdated courses.
UC15 - Monitor Platform Analytics: Admins view system statistics.
UC16 - Login: Users authenticate to access the platform.
UC17 - Register: New users create accounts.
UC18 - Generate Certificate: System automatically creates certificates when students finish everything.
UC19 - Update Analytics: System tracks student progress automatically.
UC20 - Validate Quiz Result: System checks answers and calculates scores.
